# 因子版本化 + 因子广场 — 后端实施方案

## 一、概述

将"我的因子"从单版本（覆盖式编辑）改为多版本（每次编辑生成新版本，旧版本完整保留）。同时新增"因子广场"公共表，支持研究员提交因子到广场供 PM 审阅。

---

## 二、数据库改造

### 2.1 用户私域库 — factor_versions 表（新增）

保留现有 `factor_metadata` 表不动（存储因子最新元信息），新增 `factor_versions` 表存储每个版本的完整快照。

```sql
CREATE TABLE factor_versions (
    id SERIAL PRIMARY KEY,
    factor_id VARCHAR(50) NOT NULL,         -- 如 yuyang_372
    version INTEGER NOT NULL DEFAULT 1,      -- 版本号：1, 2, 3...
    factor_code VARCHAR(100) NOT NULL,
    factor_name VARCHAR(200) NOT NULL,
    factor_name_en VARCHAR(200) DEFAULT '',
    category_l3_id INTEGER DEFAULT 0,
    expression TEXT DEFAULT '',
    description TEXT DEFAULT '',
    data_sources JSONB DEFAULT '{}',
    -- 绩效指标
    ic_mean FLOAT DEFAULT 0,
    ic_ir FLOAT DEFAULT 0,
    ic_std FLOAT DEFAULT 0,
    rank_ic_mean FLOAT DEFAULT 0,
    rank_ic_ir FLOAT DEFAULT 0,
    rank_ic_std FLOAT DEFAULT 0,
    sharpe_ratio FLOAT DEFAULT 0,
    max_drawdown FLOAT DEFAULT 0,
    turnover FLOAT DEFAULT 0,
    coverage_rate FLOAT DEFAULT 0,
    -- 回测周期
    data_start_date VARCHAR(30) DEFAULT '',
    data_end_date VARCHAR(30) DEFAULT '',
    backtest_period VARCHAR(100) DEFAULT '',
    -- 股票域
    universe_type VARCHAR(20) DEFAULT '',
    universe_id VARCHAR(50) DEFAULT '',
    universe_desc VARCHAR(500) DEFAULT '',
    -- 状态
    status VARCHAR(20) DEFAULT 'created',
    -- 时间戳
    created_at TIMESTAMP DEFAULT NOW(),
    
    UNIQUE(factor_id, version)
);
```

**逻辑**：
- `factor_metadata` 仍然保存最新版本的数据（兼容现有列表/详情接口）
- `factor_versions` 保存每个版本的完整快照
- `factor_metadata.version` 字段更新为当前最新版本号（字符串 "1.0" 改为整数或保持 "2.0" 格式）

### 2.2 factor_metadata 表改动

```sql
-- 新增字段：是否已提交到广场
ALTER TABLE factor_metadata ADD COLUMN plaza_submitted BOOLEAN DEFAULT FALSE;
-- 新增字段：最后提交到广场的版本号
ALTER TABLE factor_metadata ADD COLUMN plaza_version VARCHAR(10) DEFAULT '';
```

### 2.3 公共库 — factor_plaza 表（新增）

这张表放在**公共数据库**中（非用户私域库），所有用户的提交汇聚到一起。

```sql
CREATE TABLE factor_plaza (
    id SERIAL PRIMARY KEY,
    factor_id VARCHAR(50) NOT NULL,         -- 来源因子ID
    version INTEGER NOT NULL,                -- 提交的版本号
    factor_code VARCHAR(100) NOT NULL,
    factor_name VARCHAR(200) NOT NULL,
    factor_name_en VARCHAR(200) DEFAULT '',
    category_l1_name VARCHAR(100) DEFAULT '',
    category_l2_name VARCHAR(100) DEFAULT '',
    category_l3_name VARCHAR(100) DEFAULT '',
    expression TEXT DEFAULT '',
    description TEXT DEFAULT '',
    data_sources JSONB DEFAULT '{}',
    -- 绩效
    ic_mean FLOAT DEFAULT 0,
    ic_ir FLOAT DEFAULT 0,
    rank_ic_mean FLOAT DEFAULT 0,
    rank_ic_ir FLOAT DEFAULT 0,
    sharpe_ratio FLOAT DEFAULT 0,
    max_drawdown FLOAT DEFAULT 0,
    turnover FLOAT DEFAULT 0,
    -- 回测周期
    data_start_date VARCHAR(30) DEFAULT '',
    data_end_date VARCHAR(30) DEFAULT '',
    backtest_period VARCHAR(100) DEFAULT '',
    -- 股票域
    universe_type VARCHAR(20) DEFAULT '',
    universe_id VARCHAR(50) DEFAULT '',
    universe_desc VARCHAR(500) DEFAULT '',
    -- 提交信息
    submitted_by VARCHAR(100) NOT NULL,      -- 提交人（用户名）
    submitted_at TIMESTAMP DEFAULT NOW(),
    remark TEXT DEFAULT '',                  -- 提交备注
    -- MinIO 文件引用
    source_file_key VARCHAR(200) DEFAULT '', -- 如 yuyang_372_v2.py
    universe_file_key VARCHAR(200) DEFAULT '',-- 如 yuyang_372_v2_universe.csv
    
    UNIQUE(factor_id, version)
);
```

## 三、API 改造

### 3.1 编辑因子（版本化改造）

```
PUT /api/v1/factor/my/:id
```

**现有行为**：直接覆盖 factor_metadata 中的记录。

**新行为**：
1. 先把当前 factor_metadata 的完整数据写入 `factor_versions` 表（作为旧版本快照）
2. 然后更新 factor_metadata（覆盖为新数据）
3. factor_metadata 的 `version` 字段 +1
4. 如果有源码文件，复制一份到版本化 key：`{factor_id}_v{old_version}.py`

**首次创建因子时**：
- version 设为 1
- 同时在 factor_versions 中插入 v1 记录

**请求 Body 不变**，响应新增 version 字段：

```json
{
  "success": true,
  "message": "Factor updated successfully",
  "data": {
    "version": 3
  }
}
```

### 3.4 提交到因子广场（新增）

```
POST /api/v1/factor/my/:id/submit-plaza
X-API-Key: ak_xxx
Content-Type: application/json
```

**请求**：

```json
{
  "remark": "优化了因子逻辑，IC提升20%"
}
```

**处理逻辑**：
1. 从 factor_metadata 读取当前最新版本的完整数据
2. 获取用户名（submitted_by）
3. 读取分类名称（l1/l2/l3_name）
4. 将数据插入公共库 `factor_plaza` 表
5. 更新 factor_metadata 的 `plaza_submitted = true`，`plaza_version = 当前版本`
6. 如果有源码文件，复制 MinIO key 为 `{factor_id}_v{version}.py`（确保广场引用不受用户后续修改影响）

**响应**：

```json
{
  "success": true,
  "message": "Factor submitted to plaza successfully",
  "data": {
    "plaza_id": 15,
    "version": 2
  }
}
```

**失败情况**：
```json
{ "success": false, "error": "Factor has no performance data, please complete backtest first" }
{ "success": false, "error": "This version has already been submitted" }
```

## 四、MinIO 文件版本化策略

| 操作 | key 格式 |
|------|----------|
| 当前版本源码（用户上传时） | `{factor_id}.py` |
| 版本快照源码（编辑/提交时复制） | `{factor_id}_v{version}.py` |
| 当前版本股票域文件 | `{factor_id}_universe.csv` |
| 版本快照股票域文件 | `{factor_id}_v{version}_universe.csv` |

编辑（版本升级）时：
1. 将 `{factor_id}.py` 复制为 `{factor_id}_v{old_version}.py`
2. 用户上传新文件覆盖 `{factor_id}.py`

提交到广场时：
1. 将 `{factor_id}.py` 复制为 `{factor_id}_v{current_version}.py`（如果尚未复制）
2. plaza 表记录 `source_file_key = {factor_id}_v{version}.py`

---

## 五、对现有接口的兼容

| 接口 | 变化 |
|------|------|
| `GET /factor/my/list` | 不变，仍然从 factor_metadata 读取（永远显示最新版本） |
| `GET /factor/my/:id` | 不变，返回 factor_metadata 中的最新数据 |
| `POST /factor/my/create` | 新增逻辑：创建时同时向 factor_versions 插入 v1 记录 |
| `PUT /factor/my/:id` | 改造：旧数据存入 factor_versions → 更新 metadata → version+1 |
| `DELETE /factor/my/:id` | 新增逻辑：同时删除 factor_versions 中该因子的所有版本 |
| `POST /:id/source` | 不变 |
| `GET /:id/source` | 不变（返回当前版本源码） |

## 六、验证方法

```bash
# 1. 创建因子（应在 factor_versions 中生成 v1）
curl -X POST http://61.151.241.233:8080/api/v1/factor/my/create \
  -H "X-API-Key: ak_xxx" \
  -H "Content-Type: application/json" \
  -d '{"factor_code":"test_ver","factor_name":"版本测试","category_l3_id":1,"expression":"__py_file__"}'

# 2. 编辑因子（应在 factor_versions 中生成 v2，metadata version 变为 2）
curl -X PUT http://61.151.241.233:8080/api/v1/factor/my/yuyang_xxx \
  -H "X-API-Key: ak_xxx" \
  -H "Content-Type: application/json" \
  -d '{"factor_name":"版本测试-改名","ic_mean":0.05}'

# 3. 获取版本列表
curl http://61.151.241.233:8080/api/v1/factor/my/yuyang_xxx/versions \
  -H "X-API-Key: ak_xxx"

# 4. 获取 v1 详情
curl http://61.151.241.233:8080/api/v1/factor/my/yuyang_xxx/version/1 \
  -H "X-API-Key: ak_xxx"

# 5. 提交到广场
curl -X POST http://61.151.241.233:8080/api/v1/factor/my/yuyang_xxx/submit-plaza \
  -H "X-API-Key: ak_xxx" \
  -H "Content-Type: application/json" \
  -d '{"remark":"初版提交"}'

# 6. 查看广场列表
curl http://61.151.241.233:8080/api/v1/factor/plaza/list \
  -H "X-API-Key: ak_xxx"
```
