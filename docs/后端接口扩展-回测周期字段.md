# 因子更新接口扩展 — 支持回测周期字段

## 背景

前端新建 Py 文件因子时，需要填写回测起止日期和回测周期描述。数据库中已有 `data_start_date` 和 `data_end_date` 字段，但 `PUT /api/v1/factor/my/:id` 接口当前不支持更新这两个字段。另外需要新增一个 `backtest_period` 文本字段。

## 需要的改动

### 1. PUT 接口支持更新已有字段

在 `PUT /api/v1/factor/my/:id` 的可更新字段白名单中，新增以下两个已有字段：

| 字段 | 类型 | 说明 |
|------|------|------|
| `data_start_date` | string | 回测起始日期，格式 `YYYY-MM-DD` |
| `data_end_date` | string | 回测截止日期，格式 `YYYY-MM-DD` |

这两个字段数据库中已经存在（当前返回为空字符串），只需在 handler 的字段白名单中加上即可。

### 2. 新增字段 `backtest_period`

| 字段 | 类型 | 说明 |
|------|------|------|
| `backtest_period` | varchar(100) | 回测周期文本描述，如 "近3年"、"2020-2024"、"2022Q1-2024Q4" |

**数据库改动**：

```sql
ALTER TABLE factor_metadata ADD COLUMN backtest_period VARCHAR(100) DEFAULT '';
```

**PUT 接口**：允许更新 `backtest_period` 字段。

**GET 接口**：在因子详情和列表中返回 `backtest_period` 字段。

## 请求示例

```bash
curl -X PUT http://61.151.241.233:8080/api/v1/factor/my/yuyang_372 \
  -H "X-API-Key: ak_xxx" \
  -H "Content-Type: application/json" \
  -d '{
    "data_start_date": "2020-01-01",
    "data_end_date": "2024-12-31",
    "backtest_period": "近5年"
  }'
```

**期望响应**：

```json
{
  "success": true,
  "message": "Factor updated successfully"
}
```

## 改动范围

1. `factor_metadata` 表：新增 `backtest_period` 列
2. `my_factor.go` 的 `MyFactorUpdate` handler：字段白名单中加入 `data_start_date`、`data_end_date`、`backtest_period`
3. `my_factor.go` 的 GET 响应：返回 `backtest_period` 字段
