# 因子库后端接口扩展需求

## 背景

前端「我的因子」模块新增了 "Py 文件模式"，研究员在外部完成因子计算和回测后，需要手动将绩效指标录入系统。当前 `PUT /api/v1/factor/my/{factor_id}` 接口只允许更新元数据字段（factor_name、description、expression 等），不支持写入绩效指标。

## 需要修改的接口

### `PUT /api/v1/factor/my/{factor_id}`

**当前可更新字段**（已验证）：
- `factor_name`
- `description`
- `expression`
- `category_l3_id`（未验证，推测支持）
- `data_sources`（未验证，推测支持）

**需要新增支持的字段**：

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `ic_mean` | float | IC 均值 |
| `ic_std` | float | IC 标准差 |
| `ic_ir` | float | IC IR（信息比率） |
| `rank_ic_mean` | float | Rank IC 均值 |
| `rank_ic_std` | float | Rank IC 标准差 |
| `rank_ic_ir` | float | Rank IC IR |
| `sharpe_ratio` | float | 夏普比率 |
| `max_drawdown` | float | 最大回撤（0-1 之间的小数） |
| `turnover` | float | 换手率 |
| `status` | string | 因子状态，枚举值：`created` / `backtested` |

**请求示例**：

```json
PUT /api/v1/factor/my/yuyang_123
Content-Type: application/json
X-API-Key: ak_xxx

{
  "ic_mean": 0.035,
  "ic_ir": 0.42,
  "rank_ic_mean": 0.038,
  "rank_ic_ir": 0.45,
  "sharpe_ratio": 1.8,
  "max_drawdown": 0.12,
  "turnover": 0.35,
  "status": "backtested"
}
```

**期望响应**（和现有成功响应一致）：

```json
{
  "success": true,
  "message": "Factor updated successfully"
}
```

## 修改说明

1. 在 update handler 的"允许更新字段白名单"中，加入上述 10 个字段
2. 所有新增字段都是**可选的**，传哪个更新哪个，不传不动
3. 类型校验：float 字段只接受数字，status 只接受 `created` / `backtested` 枚举值
4. 不需要新建接口，只需要扩展现有接口的字段白名单

## 验证方法

修改完成后，可以用以下流程验证：

```bash
# 1. 创建一个测试因子
curl -X POST http://61.151.241.233:8080/api/v1/factor/my/create \
  -H "X-API-Key: ak_xxx" \
  -H "Content-Type: application/json" \
  -d '{"factor_code":"test_update_perf","factor_name":"测试","category_l3_id":1,"expression":"__py_file__"}'

# 2. 更新绩效指标（用返回的 factor_id 替换）
curl -X PUT http://61.151.241.233:8080/api/v1/factor/my/{factor_id} \
  -H "X-API-Key: ak_xxx" \
  -H "Content-Type: application/json" \
  -d '{"ic_mean":0.035,"ic_ir":0.42,"rank_ic_mean":0.038,"rank_ic_ir":0.45,"sharpe_ratio":1.8,"max_drawdown":0.12,"turnover":0.35,"status":"backtested"}'

# 期望返回: {"success":true,"message":"Factor updated successfully"}

# 3. 获取详情确认字段已更新
curl http://61.151.241.233:8080/api/v1/factor/my/{factor_id} \
  -H "X-API-Key: ak_xxx"

# 期望: 返回的 data 中 ic_mean=0.035, ic_ir=0.42 ... status=backtested

# 4. 清理
curl -X DELETE http://61.151.241.233:8080/api/v1/factor/my/{factor_id} \
  -H "X-API-Key: ak_xxx"
```
