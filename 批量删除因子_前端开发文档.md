# 批量删除因子 - 前端开发文档

## 概述

新增批量删除因子接口，允许用户一次性删除多个个人因子库中的因子。

**基础URL**: `http://localhost:8080/api/v1/factor`

**认证方式**: 请求头 `X-API-Key: <your_api_key>`

**权限要求**: 需要在 API Key 管理中勾选「因子库权限 → 批量删除因子」

---

## 接口说明

### POST `/api/v1/factor/my/batch-delete`

批量删除当前用户个人因子库中的因子。

#### 请求头

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| X-API-Key | string | 是 | 用户 API Key |

#### 请求体

```json
{
  "factor_ids": ["factor_001", "factor_002", "factor_003"]
}
```

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| factor_ids | string[] | 是 | 要删除的因子ID数组，最多500个 |

#### 成功响应 (200)

```json
{
  "success": true,
  "message": "Successfully deleted 3 factors",
  "deleted_count": 3,
  "requested": 3
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| success | bool | 是否成功 |
| message | string | 操作结果描述 |
| deleted_count | number | 实际删除的因子数量 |
| requested | number | 请求删除的因子数量 |

> **注意**: `deleted_count` 可能小于 `requested`，说明部分因子ID不存在（已被删除或ID错误），这不算失败。

#### 错误响应

**400 Bad Request** - 参数错误

```json
{
  "success": false,
  "error": "factor_ids is required and must be a non-empty array"
}
```

```json
{
  "success": false,
  "error": "factor_ids cannot be empty"
}
```

```json
{
  "success": false,
  "error": "Cannot delete more than 500 factors at once"
}
```

**401 Unauthorized** - 认证失败

```json
{
  "success": false,
  "error": "API key is required"
}
```

**403 Forbidden** - 无权限

```json
{
  "error": "Permission denied",
  "message": "Your API key does not have permission to access this resource"
}
```

**500 Internal Server Error** - 服务端错误

```json
{
  "success": false,
  "error": "Failed to delete factors"
}
```

## 前端集成示例

### 基础调用

```javascript
async function batchDeleteFactors(factorIds) {
  const response = await fetch('/api/v1/factor/my/batch-delete', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'X-API-Key': 'your_api_key'
    },
    body: JSON.stringify({ factor_ids: factorIds })
  });

  const result = await response.json();

  if (result.success) {
    console.log(`删除成功: ${result.deleted_count}/${result.requested}`);
  } else {
    console.error('删除失败:', result.error);
  }

  return result;
}
```

### 配合列表页的多选删除

```javascript
// 用户勾选多个因子后点击"批量删除"
async function handleBatchDelete(selectedFactors) {
  if (selectedFactors.length === 0) {
    message.warning('请先选择要删除的因子');
    return;
  }

  // 二次确认
  const confirmed = await Modal.confirm({
    title: '确认批量删除',
    content: `确定要删除选中的 ${selectedFactors.length} 个因子吗？此操作不可恢复。`,
  });

  if (!confirmed) return;

  const factorIds = selectedFactors.map(f => f.factor_id);
  const result = await batchDeleteFactors(factorIds);

  if (result.success) {
    message.success(`成功删除 ${result.deleted_count} 个因子`);

    // 部分删除的提示
    if (result.deleted_count < result.requested) {
      message.info(`${result.requested - result.deleted_count} 个因子未找到（可能已被删除）`);
    }

    // 刷新列表
    refreshFactorList();
  } else {
    message.error(`删除失败: ${result.error}`);
  }
}
```

## 注意事项

1. **单次上限500个**: 超过500个需要前端分批调用
2. **不可恢复**: 删除后无法撤销，建议前端加二次确认弹窗
3. **仅限个人因子库**: 只能删除当前用户自己的因子，不影响公共因子库
4. **部分成功**: 如果传入的ID中有不存在的，不会报错，`deleted_count` 会反映实际删除数量
