> 面向：前端工程师
> 依据：**真实接口返回**（前端工程师用真实 API Key 拉取 `GET /task/{id}/result` 核实）+ 引擎源码交叉验证

> - 落位：`backtest-engine/src/main.rs` L1574-1577（`summary.report`）、L2997（`summary.admission_report`）

## 0. 核心结论：追加式改造（不是重写）

经真实接口核实，`summary` 顶层**新旧结构共存**——旧字段仍在返回，`report` 是**新增**：

实测 `summary` keys：
```
cost_assumptions, data_alignment, engine_version, experiment_management,
factor_collinearity, factor_lifecycle, factor_orthogonality, generalization,
governance, headline_metric_basis, holdout_validation, mode_budget,
net_headline, report, research_mode, result_quality, stage_timings,
universe_point_in_time
```

| 维度 | 现状 | 改造方式 |
| --- | --- | --- |
| 旧字段 `mode_budget`/`holdout_validation`/`generalization` 等 | **仍在返回**，前端现有渲染不会白屏 | 保留，不动 |
| `summary.report`（`summary_cards`/`charts`/`warnings`/`tables`） | v0.2.6 新增 | **追加**新渲染 |
| `summary.admission_report` | 仅 admission 模式 | 追加入库看板 |

> **改造原则：不推倒重写。保留现有能取到数的旧展示，新增 `report` 驱动的卡片/图表/警告/表格 + admission 看板。所有新字段一律"有才渲染、无则降级、绝不因缺字段报错"。**

**待引擎/网关工程师确认**：旧结构（mode_budget 等）是过渡期兼容保留、未来会删，还是长期保留？若会删需给时间点——这决定是否保留旧渲染兜底。

## 1. 取数路径（唯一入口）

`GET /api/v1/backtest/task/{task_id}/result` 返回 `data.summary`，其中：

```
data.summary.report            → BacktestReport（所有模式都有，实测确认）
data.summary.admission_report  → 仅 research_mode=admission 时存在（待样本验证）
data.factor_results[]          → 单因子任务实测无 variant 字段；仅中性化任务可能有（待样本验证）
```

前端从 `summary.report` 渲染新主结果，`summary.admission_report` 渲染入库看板，旧 `summary.*` 保留现渲染。

## 2. 提交页改造（`Backtest/SubmitContent.vue` + `MyFactors.vue`）

### 2.1 新增费率三件套（`backtest_params` 下）

网关已加字段：`risk_free_rate` / `buy_cost_bps` / `sell_cost_bps`（单位 bp）。

- 研究模式（quick/research）：提供三个数值输入框，可留空（留空引擎回退 A 股拆分模型）
- **admission 模式：置灰并显示固定值**——买入 5bp / 卖出 10bp（引擎强制覆盖，前端传了也不生效）

### 2.2 admission 模式的强制覆盖提示（防误用）

`research_mode=admission` 时，用户填的 `universe` / `forward_periods` / 费率**都会被引擎强制覆盖**。提交前应提示：

> 「入库审核模式将强制：股票池=中证1000、IC周期=5日、样本内外固定切片、费率买5/卖10bp。以上配置将被忽略。」

### 2.3 其余透传字段

`parameter_scan` / `udfs` / `field_mappings` / `aggregation` / `memory_limit_mb` 网关已透传，如需 UI 再加；不加也不影响提交。

## 3. 结果页改造（`Backtest/ResultContent.vue` 追加式，新增取数层）

> 全部为追加渲染，不动旧 `summary.*` 展示。以下新字段实测均可能缺失，一律"有才渲染、无则降级"。

### 3.1 summary_cards 渲染（grade/reference 为可选）

遍历 `summary.report.summary_cards[]`。实测 card 只稳定含 `key`/`label`/`value`/`unit`/`metadata`，**`grade`/`reference` 实测缺失**（源码 `skip_serializing_if=none`）：

- `value` 按 `unit` 格式化：`pct`→百分比、`ratio`→保留2位、`days`/`count`→整数
- **`grade` 有才着色**：`excellent`绿 / `good`蓝 / `fair`黄 / `poor`红 / `na`灰；无则不着色
- **`reference` 有才作 tooltip**；无则不显示
- ⚠️ 待引擎确认：grade/reference 是所有模式都不返回，还是仅 research/admission 才有？

### 3.2 warnings 渲染（suggestion 为可选）

遍历 `summary.report.warnings[]`。实测 warning 只含 `code`/`message`/`severity`（小写 `info`/`warning`/`critical`），**`suggestion` 实测缺失**：

- 按 severity 排序：critical → warning → info
- `critical` 红色置顶
- **`suggestion` 有才渲染**为「建议行动」区块；无则只显示 message
- ⚠️ 待引擎确认：suggestion 是未实现还是仅特定 warning 才有？

### 3.3 charts 分发渲染（16 种 type，quick 实测仅 7 种）

`summary.report.charts[]` 是 tagged enum，按 `type` switch 分发到对应 ECharts 组件。**16 种全做分发，未识别 type 静默跳过**。

**实测 quick 模式只产出以下 7 种**（可自测）：
`ic_time_series` / `rolling_ic` / `nav_and_drawdown` / `quantile_cumulative_nav` / `annual_monthly_heatmap` / `turnover_and_cost` / `data_quality_coverage`

**其余 9 种 quick 下不产出**（需样本 task_id 才能核对字段）：
`cost_sensitivity` / `capacity_curve` / `bucket_ic_heatmap` / `alpha_decay` / `autocorrelation_curve` / `nonlinear_response` / `turnover_heatmap` / `brinson_attribution` / `exposure_attribution`

全部 16 种字段（严格按 schema.rs，先按此写好分发，9 种待样本核对）：

| --- | --- | --- |

## 4. 入库审核看板（`summary.admission_report`，仅 admission 模式，**待样本验证**）

⚠️ 已核的 40+ 任务全为 quick 模式，`admission_report` 未在真实接口中出现过，以下结构来自源码 `admission/mod.rs`，**尚未用真实数据验证**。

结构（源码）：`split_date` / `in_sample{ic_mean,icir,rank_ic_mean,excess_annual_return,turnover_mean,sharpe,max_drawdown,n_days}` / `out_of_sample{同}` / `wq_score` / `wq_components{ir,excess_annual_return,turnover}` / `pool` / `ic_period` / `pool_override`。

渲染要点：

- `wq_score` 作「一句话评分」置顶：>0.3 绿 / 0~0.3 黄 / <0 红
- `in_sample` vs `out_of_sample` 并排对比，样本外明显劣化时高亮预警（过拟合信号）
- `pool_override=true` 时提示「用户股票池已被强制覆盖为中证1000」
- 任一指标为 `null`（样本不足致 NaN）时显示「样本不足」而非 0

⚠️ 待引擎提供：一个 admission 模式 task_id，供核对 `admission_report` 真实结构。

## 5. variant 分组（**实测不存在，待样本验证**）

⚠️ 单因子任务实测 `factor_results` 长度为 1，**无 `variant` 字段**，keys 为：
`factor_name / ic_mean / ic_std / ic_ir / rank_ic_mean / rank_ic_std / rank_ic_ir / period_ic_stats / annual_return / annual_volatility / sharpe_ratio / max_drawdown / win_rate / turnover_mean / layer_returns / monotonicity`。已核 40+ 任务无一多 variant。

源码设计中 variant 应为 `raw` / `neutral_selected` / `neutral_all` / 最多 20 个 `neutral_each_*`，**但生产接口尚未见到**。

前端处理：

- **variant 字段有才分组**（`factor_results[].variant`）；无则按单结果渲染（当前实际情况）
- `neutral_each_*` 有才折叠为「单因子剥离明细」表格
- 前端现有 variant Tab 逻辑（`ResultContent.vue` L437-445 读 `neutralization.variants`）**先不动**，等确认 variant 真实位置后再改

⚠️ 待引擎提供：一个跑过中性化的多 variant task_id，供确认 variant 字段到底在 `factor_results[].variant` 还是别处。

1. **追加式改造**：不动现有能取到数的旧展示，新增 `report`（summary_cards/charts/warnings/tables）+ admission 看板。风险最低。
2. **所有新字段按"可选"处理**：`grade`/`reference`/`suggestion`/`variant` 有才渲染，无则降级，绝不因缺字段报错。
