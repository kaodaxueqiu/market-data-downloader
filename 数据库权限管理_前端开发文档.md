# 数据库权限管理 - 前端开发文档

## 一、功能概述

本功能用于在前端管理 **PostgreSQL** 和 **ClickHouse** 两个数据库的用户与权限。管理员可以：

- 查看数据库列表、表列表
- 管理数据库用户（创建、删除、改密码）
- 管理用户权限（查看、授予、撤销，支持数据库级/表级/操作级）
- 查看用户与 API Key 的绑定关系

## 二、菜单与路由

| 菜单层级 | 名称 | 菜单 ID | 路由路径 |
|---------|------|---------|---------|
| 一级 | 控制中心 | `control_center` | `/control-center` |
| 二级 | 数据库管理 | `db_management` | `/control-center/db-management` |

## 三、认证方式

所有接口需要 **Admin 认证**，请求头中需携带：

```
X-API-Key: <管理员的API Key>
```

或使用 Master Key：

```
X-Master-Key: master_key_2025
```

## 四、接口基础路径

```
/api/v1/admin/db-permissions
```

## 五、接口详细说明

### 5.1 列出数据库

查询 PostgreSQL 和 ClickHouse 中的所有数据库，用于页面左侧的数据库树形导航。

```
GET /api/v1/admin/db-permissions/databases?db_type={all|postgresql|clickhouse}
```

**Query 参数：**

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|-------|------|
| db_type | string | 否 | all | 筛选数据库类型 |

**响应示例：**

```json
{
  "success": true,
  "data": {
    "postgresql": {
      "databases": [
        { "name": "factor_db", "db_type": "postgresql" },
        { "name": "finance_db", "db_type": "postgresql" }
      ],
      "connection_info": {
        "host": "192.168.30.11",
        "port": 5432,
        "grant_databases": ["factor_db", "finance_db"]
      }
    },
    "clickhouse": {
      "databases": [
        { "name": "market_data", "db_type": "clickhouse" },
        { "name": "market_mart", "db_type": "clickhouse" }
      ],
      "connection_info": {
        "host": "192.168.30.10",
        "port": 9000,
        "grant_databases": ["market_data", "market_mart"]
      }
    }
  }
}
```

### 5.2 列出数据库表

查询指定数据库中的所有表，用于权限配置时的表选择。

```
GET /api/v1/admin/db-permissions/databases/{db_type}/{db_name}/tables
```

**Path 参数：**

| 参数 | 类型 | 说明 |
|------|------|------|
| db_type | string | `postgresql` 或 `clickhouse` |
| db_name | string | 数据库名称，如 `factor_db` |

**响应示例：**

```json
{
  "success": true,
  "data": {
    "db_type": "clickhouse",
    "database": "market_data",
    "tables": [
      {
        "name": "kline_1min",
        "database": "market_data",
        "db_type": "clickhouse",
        "engine": "MergeTree",
        "rows": 1520000000,
        "comment": "1分钟K线数据"
      }
    ],
    "total": 59
  }
}
```

**字段说明：**

| 字段 | 说明 |
|------|------|
| name | 表名 |
| engine | 存储引擎（ClickHouse 独有） |
| rows | 行数估算 |
| comment | 表注释 |

```
GET /api/v1/admin/db-permissions/users?db_type={all|postgresql|clickhouse}
```

```json
{
  "success": true,
  "data": {
    "postgresql": {
      "users": [
        { "username": "researcher_zhang", "db_type": "postgresql", "can_login": true, "created_at": "" },

      ],
      "total": 2
    },
    "clickhouse": {
      "users": [
        { "username": "researcher_zhang", "db_type": "clickhouse", "can_login": true }
      ],
      "total": 1
    },
    "apikey_bindings": [
      {
        "api_key_name": "张三",
        "api_key": "zz_abc1****",
        "postgresql_username": "researcher_zhang",

    ]
  }

```

> **前端展示建议：** 用户列表中，如果用户在 `apikey_bindings` 中出现，显示一个「已绑定 API Key」的标签，点击可查看绑定的 API Key 名称。

### 5.4 创建数据库用户

创建一个独立的数据库用户（不绑定 API Key）。

```
POST /api/v1/admin/db-permissions/users
```

**请求体：**

```json
{
  "db_type": "both",
  "username": "new_researcher",
  "password": "SecurePass123!"
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| db_type | string | 是 | `postgresql`、`clickhouse` 或 `both`（两个库都创建） |
| username | string | 是 | 用户名（字母开头，可含字母/数字/下划线） |
| password | string | 是 | 密码 |

**用户名规则：**

| 数据库 | 规则 |
|--------|------|
| PostgreSQL | 字母开头，可含字母/数字/下划线，1-63 字符 |
| ClickHouse | 字母或下划线开头，可含字母/数字/下划线/连字符，2-128 字符 |

**保留用户名（不可使用）：**
- PostgreSQL：`postgres`、`admin`、`root`、`public`、`pg_` 开头
- ClickHouse：`default`、`admin`、`root`、`system`

**成功响应：**

```json
{
  "success": true,
  "message": "数据库用户创建成功",
  "data": { "db_type": "both", "username": "new_researcher" }
}
```

**错误响应（校验失败）：**

```json
{
  "error": "用户名 'admin' 是保留关键字，不能使用",
  "field": "postgresql_username"
}
```

---

### 5.5 删除数据库用户

```
DELETE /api/v1/admin/db-permissions/users/{username}?db_type={postgresql|clickhouse|both}
```

| 参数 | 位置 | 必填 | 默认值 | 说明 |
|------|------|------|-------|------|
| username | path | 是 | - | 用户名 |
| db_type | query | 否 | both | 从哪个数据库删除 |

**成功响应：**

```json
{
  "success": true,
  "message": "数据库用户删除成功",
  "data": { "username": "old_user", "db_type": "both" }
}
```

> **前端注意：** 删除前应弹出二次确认对话框，提示「此操作将永久删除该用户及其所有权限，不可恢复」。如果用户绑定了 API Key，建议提示「该用户已绑定 API Key [xxx]，删除后 API Key 的数据库配置将失效」。

### 5.7 查看用户权限

获取指定用户在各数据库/表上的权限详情，用于权限展示和编辑页面。

```
GET /api/v1/admin/db-permissions/users/{username}/grants?db_type={all|postgresql|clickhouse}
```

**响应示例：**

```json
{
  "success": true,
  "data": {
    "username": "researcher_zhang",
    "grants": {
      "postgresql": {
        "username": "researcher_zhang",
        "db_type": "postgresql",
        "grants": [
          {
            "db_type": "postgresql",
            "database": "factor_db",
            "schema": "public",
            "table": "*",
            "privileges": ["SELECT"]
          },
          {
            "db_type": "postgresql",
            "database": "finance_db",
            "schema": "public",
            "table": "balance_sheet",
            "privileges": ["SELECT", "INSERT"]
          }
        ]
      },
      "clickhouse": {
        "username": "researcher_zhang",
        "db_type": "clickhouse",
        "grants": [
          {
            "db_type": "clickhouse",
            "database": "market_data",
            "table": "*",
            "privileges": ["SELECT"]
          },
          {
            "db_type": "clickhouse",
            "database": "*",
            "table": "*",
            "privileges": ["SHOW"]
          }
        ]
      }
    }
  }
}
```

**Grant 对象字段：**

| 字段 | 类型 | 说明 |
|------|------|------|
| db_type | string | `postgresql` 或 `clickhouse` |
| database | string | 数据库名，`*` 表示所有 |
| schema | string | Schema（仅 PostgreSQL，通常是 `public`） |
| table | string | 表名，`*` 表示该数据库下所有表 |
| privileges | string[] | 权限列表，如 `["SELECT", "INSERT"]` |

**支持的权限类型：**

| 权限 | 说明 |
|------|------|
| SELECT | 查询数据 |
| INSERT | 插入数据 |
| UPDATE | 更新数据 |
| DELETE | 删除数据 |
| CREATE | 创建表 |
| DROP | 删除表 |
| ALTER | 修改表结构 |
| TRUNCATE | 清空表 |
| SHOW | 查看表结构（ClickHouse） |

> **注意**：PostgreSQL 的 `CONNECT` 权限默认所有用户都有（与 ClickHouse 行为一致），后端不管理也不返回此权限。前端无需关注 CONNECT。

### 5.8 授予权限

为用户授予指定数据库/表上的权限。

```
POST /api/v1/admin/db-permissions/users/{username}/grants
```

**请求体：**

```json
{
  "db_type": "clickhouse",
  "database": "market_data",
  "tables": ["kline_1min", "kline_5min"],
  "privileges": ["SELECT", "INSERT"]
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| db_type | string | 是 | `postgresql` 或 `clickhouse` |
| database | string | 是 | 数据库名 |
| tables | string[] | 是 | 表名列表，`["*"]` 表示所有表 |
| privileges | string[] | 是 | 权限列表 |

**示例 - 授予整个数据库的只读权限：**

```json
{
  "db_type": "postgresql",
  "database": "factor_db",
  "tables": ["*"],
  "privileges": ["SELECT"]
}
```

**成功响应：**

```json
{
  "success": true,
  "message": "权限授予成功",
  "data": {
    "username": "researcher_zhang",
    "db_type": "clickhouse",
    "database": "market_data",
    "tables": ["kline_1min", "kline_5min"],
    "privileges": ["SELECT", "INSERT"]
  }
}
```

### 5.9 撤销权限

撤销用户在指定数据库/表上的权限。

```
DELETE /api/v1/admin/db-permissions/users/{username}/grants
```

**请求体格式同 5.8**（需要在 DELETE 请求中携带 Body）。

**示例 - 撤销某用户在 market_data 库 kline_1min 表的 INSERT 权限：**

```json
{
  "db_type": "clickhouse",
  "database": "market_data",
  "tables": ["kline_1min"],
  "privileges": ["INSERT"]
}
```

**成功响应：**

```json
{
  "success": true,
  "message": "权限撤销成功",
  "data": {
    "username": "researcher_zhang",
    "db_type": "clickhouse",
    "database": "market_data",
    "tables": ["kline_1min"],
    "privileges": ["INSERT"]
  }
}
```

---

### 5.10 撤销所有权限

一键撤销用户在指定数据库上的全部权限。

```
DELETE /api/v1/admin/db-permissions/users/{username}/grants/all
```

**请求体：**

```json
{

}
```

**成功响应：**


### 5.11 批量授权

一次为用户授予多个数据库/表的权限。

```
POST /api/v1/admin/db-permissions/users/{username}/grants/batch
```

**请求体：**

```json
{
  "grants": [
    {
      "db_type": "postgresql",
      "database": "factor_db",
      "tables": ["*"],
      "privileges": ["SELECT"]
    },
    {
      "db_type": "clickhouse",
      "database": "market_data",
      "tables": ["*"],
      "privileges": ["SELECT"]
    },
    {
      "db_type": "clickhouse",
      "database": "market_mart",
      "tables": ["*"],
      "privileges": ["SELECT"]
    }
  ]
}
```

**响应示例：**

```json
{
  "success": true,
  "message": "成功 3 项，失败 0 项",
  "data": {
    "successes": 3,
    "errors": []
  }
}
```

**部分失败的响应：**

```json
{
  "success": false,
  "message": "成功 2 项，失败 1 项",
  "data": {
    "successes": 2,
    "errors": ["授权失败 (target=market_data.nonexistent_table): ..."]
  }
}
```

## 六、错误响应格式

所有接口的错误响应统一格式：

```json
{
  "error": "错误描述信息"
}
```

校验错误会额外返回 `field` 字段：

```json
{
  "error": "用户名必须以字母开头，只能包含字母、数字和下划线",
  "field": "postgresql_username"
}
```

**HTTP 状态码：**

| 状态码 | 含义 |
|--------|------|
| 200 | 成功 |
| 400 | 请求参数错误 / 校验失败 |
| 401 | 未认证 |
| 403 | 无权限 |
| 500 | 服务端错误 |
| 503 | 数据库用户管理未启用 |

## 七、权限注册表

本模块对应的权限分组 ID 为 `db_permission_api`，前端权限管理 Tab 名称：**数据库权限**。

| 权限 ID | 名称 | 风险级别 |
|---------|------|---------|
| admin.dbperm.databases.list | 列出数据库 | low |
| admin.dbperm.tables.list | 列出数据库表 | low |
| admin.dbperm.users.list | 列出数据库用户 | medium |
| admin.dbperm.users.create | 创建数据库用户 | high |
| admin.dbperm.users.delete | 删除数据库用户 | critical |
| admin.dbperm.users.password | 修改密码 | high |
| admin.dbperm.grants.view | 查看用户权限 | low |
| admin.dbperm.grants.grant | 授予权限 | critical |
| admin.dbperm.grants.revoke | 撤销权限 | critical |
| admin.dbperm.grants.revoke_all | 撤销所有权限 | critical |
| admin.dbperm.grants.batch | 批量授权 | critical |

**快捷权限组：**
- `dbperm_readonly` — 只读查看（列出数据库/表/用户/权限）
- `dbperm_full` — 完整管理

### 8.4 交互流程

```
进入页面
  │
  ├─ 调用 GET /databases 加载左侧数据库树
  ├─ 调用 GET /users 加载用户列表
  │
  ├─ 点击用户「查看权限」
  │    └─ 调用 GET /users/{username}/grants 展示权限详情
  │
  ├─ 点击「权限矩阵」Tab，选择用户+数据库
  │    ├─ 调用 GET /databases/{type}/{name}/tables 加载表列表
  │    ├─ 调用 GET /users/{username}/grants 加载现有权限
  │    └─ 用户勾选后点击保存
  │         └─ 调用 POST /users/{username}/grants/batch 批量授权
  │            或 DELETE /users/{username}/grants 撤销权限
  │
  ├─ 点击「创建用户」
  │    └─ 弹窗填写信息 → POST /users
  │
  └─ 点击「删除用户」
       └─ 二次确认 → DELETE /users/{username}
```

## 十、数据中心可见性控制（重要变更）

### 变更说明

**旧方案**（已废弃）：数据中心的 4 个数据源 tab 可见性由 API Key 管理中的 `datasource_access` 字段控制（勾选开关）。

**新方案**：数据中心可见性完全由用户绑定的**数据库账号的实际权限**驱动，与数据库权限管理统一归口。

### 判断规则

| 数据源 tab | 对应数据库 | 可见条件 |
|---|---|---|
| 财务数据库 (postgresql) | PG `finance_db` | 用户的 PG 账号在 finance_db 有任意表级权限 |
| 数据加工库 (clickhouse) | CH `market_mart` | 用户的 CH 账号在 market_mart 有任意权限 |
| 行情镜像库 (clickhouse_data) | CH `market_data` | 用户的 CH 账号在 market_data 有任意权限 |
| 实时行情库 (redis) | Redis | 始终可见，不做权限控制 |

### 接口返回变化

`GET /api/v1/dbdict/datasources` 返回的 `has_permission` 和 `available` 字段现在基于数据库账号的实际权限，不再依赖 `datasource_access`。

### 前端需配合的改动

1. **移除** API Key 管理页面中的"数据源权限"勾选 UI（4 个数据源的开关）
2. 数据中心渲染 tab 时，根据 `/datasources` 返回的 `available` 字段决定是否显示
3. 权限管理统一在"数据库管理"页面操作，修改数据库权限后数据中心可见性自动生效
